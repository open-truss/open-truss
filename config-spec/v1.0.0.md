Open Truss uses nested composable configurations to define everything in the system. This allows for its users to share and build on each other's work and for the community to help create reusable configurations.

# Config Spec v1

While developed with React in mind (see [@open-truss/open-truss on NPM](https://www.npmjs.com/package/@open-truss/open-truss)), this specification is not tied to React.

## Key concepts

- **version** – must be `1`. Determines which engine renders the workflow.
- **frames** – an ordered list of frame objects.
  - **view.component** – the name of a component.
  - **view.props** – an optional map of prop names to values. Values can be:
    - Literal YAML values (strings, numbers, booleans, arrays, objects)
    - `:signalName` – wires a declared signal to the prop
  - **data** – optional data configuration passed to the component as the `data` prop.
  - **frames** – optional nested sub-frames, rendered as `children` of the parent component.
- **Components** – the component to render, e.g. a React component. Open Truss components should be registered into an allowlist.
- **signals** – a map of signal names to signal type strings. Signals are reactive values that can be shared across components within a workflow.
- **renderFrames** – controls how child frames are displayed:
  - `all` (default) – renders all sub-frames simultaneously as siblings.
  - `inSequence` – renders one frame at a time with next/back navigation signals.
- **frameWrapper** – optional name of a registered component that wraps every rendered frame.

## Schema

```yaml
workflow:
  version: 1                 # required, positive integer
  id: my-workflow            # optional, used for cursor persistence
  debug: false               # optional, enables debug mode
  signals:                   # optional
    signalName: signalType   # e.g. integer: number
  frameWrapper: MyWrapper    # optional component name
  renderFrames:              # optional
    type: all                # or: inSequence
    next: :nextSignal        # required for inSequence
    back: :backSignal        # optional, for inSequence
  frames:                    # required, at least one
    - frame:
      view:
        component: ComponentName
        props:               # optional
          title: Hello
          link: <NextLink />
          count: :integer
      data:                  # optional
      frames:                # optional nested sub-frames
        - frame:
          view:
            component: NestedComponent
```

## Example

```yaml
workflow:
  version: 1
  id: integer-squares
  signals:
    integer: number
    runCalculation: NavigateFrame
  renderFrames:
    type: inSequence
    next: :runCalculation
  frames:
    - frame:
      view:
        component: IntegerInput
        props:
          submit: :runCalculation
    - frame:
      view:
        component: ShowIntegerSquared

# The same workflow with nested frames:
---
workflow:
  version: 1
  frames:
    - frame:
      view:
        component: YourAppExampleComponent
      frames:
        - frame:
          view:
            component: OTExampleComponent
```
